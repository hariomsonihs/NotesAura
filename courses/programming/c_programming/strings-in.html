Strings in C Programming Language

Introduction to Strings

Strings in C are sequences of characters stored as arrays of char, terminated by a null character (\0).

  • Definition: A string is an array of characters ending with a null terminator, used to represent text.
  • Key Points:
    • Strings are not a built-in data type but are implemented as char arrays or pointers.
    • Manipulated using standard library functions from <string.h>.
    • The null character (\0) marks the end of a string, essential for processing.

Declaring and Initializing Strings

Strings can be declared as character arrays or pointers and initialized in various ways.

  • Definition: String declaration involves defining a char array or pointer to hold the string data.
  • Key Points:
    • Array Declaration: char str[size] allocates fixed memory.
    • Pointer Declaration: char *str points to a string literal or dynamic memory.
    • String literals (e.g., "Hello") are stored in read-only memory.
    • Always ensure enough space for the null terminator.

Example: Declaring and Initializing Strings

#include <stdio.h>

int main() {
    char str1[10] = "Hello"; // Array initialization
    char str2[] = "World"; // Auto-sized array
    char *str3 = "C Programming"; // Pointer to string literal
    printf("Str1: %s, Str2: %s, Str3: %s\\n", str1, str2, str3);
    return 0;
}
  • char str1[10]: Fixed-size array with space for null terminator.
  • char str2[]: Compiler auto-sizes based on string length.
  • char *str3: Points to a string literal in read-only memory.
Note: Modifying string literals (e.g., str3[0] = 'X') causes undefined behavior as they are stored in read-only memory.

Input and Output of Strings

Strings are handled using standard input/output functions like printf, scanf, and others.

  • Definition: Input/output operations read or display strings using standard library functions.
  • Key Points:
    • printf: Outputs strings with %s format specifier.
    • scanf: Reads strings, stops at whitespace, requires array size to avoid overflow.
    • gets/fgets: Reads entire lines, fgets is safer as it limits input size.
    • puts/fputs: Outputs strings, puts adds a newline.

Example: String Input/Output

#include <stdio.h>

int main() {
    char name[20];
    printf("Enter your name: ");
    fgets(name, sizeof(name), stdin); // Safe input
    printf("Hello, %s", name);
    return 0;
}
  • fgets: Reads input safely, including spaces, up to specified size.
  • printf: Displays string with %s.

String Manipulation Functions

C provides a rich set of string manipulation functions in <string.h>.

  • Definition: Functions for manipulating strings, such as copying, concatenating, and comparing.
  • Key Points:
    • strlen: Returns string length (excluding \0).
    • strcpy/strncpy: Copies strings, strncpy is safer with size limit.
    • strcat/strncat: Concatenates strings, strncat limits appended characters.
    • strcmp/strncmp: Compares strings, returns 0 if equal.
    • strchr/strrchr: Finds first/last occurrence of a character.
    • strstr: Finds substring in a string.

Example: String Manipulation

#include <stdio.h>
#include <string.h>

int main() {
    char str1[20] = "Hello";
    char str2[20] = "World";
    printf("Length of str1: %zu\\n", strlen(str1));
    strcat(str1, " ");
    strcat(str1, str2); // Concatenate
    printf("Concatenated: %s\\n", str1);
    printf("Comparison: %d\\n", strcmp(str1, "Hello World"));
    return 0;
}
  • strlen: Calculates length of "Hello" (5).
  • strcat: Appends strings to form "Hello World".
  • strcmp: Compares strings, returns 0 for equality.

Character Arrays vs. String Literals

Character arrays and string literals differ in mutability and memory allocation.

  • Definition: Character arrays are mutable, while string literals are immutable and stored in read-only memory.
  • Key Points:
    • Character arrays allow modification of individual characters.
    • String literals are constant and cannot be modified.
    • Arrays require explicit size; literals are handled by the compiler.

Example: Character Arrays vs. String Literals

#include <stdio.h>

int main() {
    char str1[] = "Mutable"; // Character array
    char *str2 = "Immutable"; // String literal
    str1[0] = 'm'; // Valid
    printf("Array: %s\\n", str1);
    // str2[0] = 'i'; // Undefined behavior
    return 0;
}
  • char str1[]: Mutable array, can be modified.
  • char *str2: Points to immutable string literal.

Dynamic Memory Allocation for Strings

Strings can be allocated dynamically using malloc or calloc for flexible sizes.

  • Definition: Dynamic allocation creates strings with sizes determined at runtime.
  • Key Points:
    • Use malloc or calloc to allocate memory.
    • Always free allocated memory using free to avoid memory leaks.
    • Ensure space for null terminator in allocation.

Example: Dynamic Strings

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    char *str = malloc(20 * sizeof(char));
    if (str == NULL) {
        printf("Memory allocation failed\\n");
        return 1;
    }
    strcpy(str, "Dynamic String");
    printf("Dynamic: %s\\n", str);
    free(str); // Free memory
    return 0;
}
  • malloc: Allocates memory for string.
  • free: Releases allocated memory.

Advanced String Operations

Advanced string handling includes parsing, tokenization, and format conversion.

  • Definition: Advanced operations involve complex string manipulations like splitting or converting data.
  • Key Points:
  • strtok: Tokenizes strings by splitting on delimiters.
  • sscanf/sprintf: Reads/formats strings with specific data types.
  • Wide Strings: Use wchar_t for Unicode/multibyte characters (C99).

Example: Tokenization and Formatting

#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "apple,banana,orange";
    char *token = strtok(str, ",");
    while (token != NULL) {
        printf("Token: %s\\n", token);
        token = strtok(NULL, ",");
    }
    char buffer[50];
    sprintf(buffer, "Number: %d, Float: %.2f", 42, 3.14);
    printf("Formatted: %s\\n", buffer);
    return 0;
}
  • strtok: Splits string into tokens based on commas.
  • sprintf: Formats data into a string.

Wide Strings and Multibyte Characters

Wide strings support Unicode and multibyte characters for internationalization.

  • Definition: Wide strings use wchar_t to handle extended character sets.
  • Key Points:
    • Declared with wchar_t and prefixed with L (e.g., L"Hello").
    • Use <wchar.h> functions like wcslen, wcscpy.
    • Supports UTF-16/UTF-32 depending on platform.
