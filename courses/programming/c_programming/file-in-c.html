File Handling in C

What is File Handling?

File handling in C allows us to store data permanently in files instead of temporary memory (RAM). It uses functions from <stdio.h> like fopen, fclose, fgetc, fputc, fgets, fputs, fprintf, and fscanf.

  • Text Files: Store data in human-readable format.
  • Binary Files: Store data in machine-readable format.

File Modes in C

  • "r" – Read only (file must exist).
  • "w" – Write only (creates new file, erases if exists).
  • "a" – Append (writes at end of file).
  • "r+" – Read & write (file must exist).
  • "w+" – Read & write (creates new file, erases if exists).
  • "a+" – Read & append (reads & writes at end).

File Handling Process

Open File (fopen)
Read/Write Data
Close File (fclose)

Example 1: Writing to a File

#include <stdio.h>

int main() {
    FILE *fptr;
    fptr = fopen("example.txt", "w");

    if (fptr == NULL) {
        printf("Error opening file!");
        return 1;
    }

    fprintf(fptr, "Hello, File Handling in C!\n");
    fclose(fptr);

    printf("Data written successfully.\n");
    return 0;
}

Example 2: Reading from a File

#include <stdio.h>

int main() {
    FILE *fptr;
    char ch;

    fptr = fopen("example.txt", "r");

    if (fptr == NULL) {
        printf("File not found!");
        return 1;
    }

    while ((ch = fgetc(fptr)) != EOF) {
        putchar(ch);
    }

    fclose(fptr);
    return 0;
}

Example 3: Appending to a File

#include <stdio.h>

int main() {
    FILE *fptr;
    fptr = fopen("example.txt", "a");

    if (fptr == NULL) {
        printf("Error opening file!");
        return 1;
    }

    fprintf(fptr, "Appending new line to file.\n");
    fclose(fptr);

    printf("Data appended successfully.\n");
    return 0;
}

Common Pitfalls

  • Forgetting to close files → memory leaks.
  • Opening a non-existing file in "r" mode → error.
  • Using wrong mode (e.g., writing in read-only mode).
  • File paths must be correct (absolute/relative).
Conclusion:

File handling in C is essential for storing and retrieving data permanently. Mastering fopen, fclose, fprintf, fscanf, and other functions allows efficient text and binary file operations. Always handle errors and close files properly.
