Memory in C Programming

Introduction

Memory in C refers to how data is stored, allocated, and managed during program execution. Understanding memory helps in writing efficient, bug-free, and optimized C programs.

Memory Layout of a C Program

  • Code Segment: Stores compiled machine code instructions (read-only).
  • Data Segment: Stores global and static variables.
    • Initialized Data: Variables with assigned values.
    • Uninitialized Data (BSS): Variables without assigned values.
  • Heap: Memory for dynamic allocation using malloc(), calloc(), realloc(), free().
  • Stack: Stores local variables, function calls, return addresses. Follows LIFO principle.

C Program Memory Layout

Code
Data (Init + BSS)
Heap (grows up)
Stack (grows down)

Heap ↑ grows upward, Stack ↓ grows downward

Static vs Dynamic Memory

  • Static Memory Allocation: Size fixed at compile-time. Examples: global, static, local variables.
  • Dynamic Memory Allocation: Memory allocated at runtime using standard library functions from <stdlib.h>.

Dynamic Memory Allocation Functions

  • malloc(size): Allocates block of memory (uninitialized).
  • calloc(n, size): Allocates memory for array, initializes to 0.
  • realloc(ptr, newSize): Resizes previously allocated memory.
  • free(ptr): Releases memory back to system.

Example: Using malloc() and free()

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr;
    arr = (int*) malloc(5 * sizeof(int));

    if(arr == NULL) {
        printf("Memory not allocated!\n");
        return 1;
    }

    for(int i = 0; i < 5; i++) {
        arr[i] = i * 2;
    }

    for(int i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }

    free(arr); // release memory
    return 0;
}

Common Memory Issues

  • Memory Leak: Forgetting to free allocated memory.
  • Dangling Pointer: Accessing freed memory.
  • Buffer Overflow: Writing outside allocated memory.
  • Segmentation Fault: Invalid memory access.
Pro Tip: Always free() dynamically allocated memory, and set pointer to NULL after freeing.
Conclusion:

Memory management in C is powerful but requires care. Understanding stack, heap, and allocation functions ensures efficient and bug-free programs. Mastery of memory handling helps avoid leaks, crashes, and improves program performance.
