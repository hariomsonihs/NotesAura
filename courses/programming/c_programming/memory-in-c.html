Memory Management in C

Introduction

Memory management in C is the process of efficiently allocating, using, and freeing memory during program execution. C gives direct access to memory through pointers, which makes it powerful but also error-prone if not handled properly.

Memory Layout in C

  • Code Segment: Stores compiled instructions.
  • Data Segment: Stores global and static variables.
  • Stack: Stores function calls, local variables.
  • Heap: Stores dynamically allocated memory.

Memory Layout Diagram

[ Code ] → [ Data ] → [ Heap ] → [ Stack (grows downward) ]

Dynamic Memory Allocation Functions

  • malloc(size): Allocates memory of given size (uninitialized).
  • calloc(n, size): Allocates memory for array of n elements (initialized to 0).
  • realloc(ptr, size): Resizes allocated memory.
  • free(ptr): Frees previously allocated memory.

Example: Using malloc and free

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    ptr = (int*) malloc(5 * sizeof(int));

    if (ptr == NULL) {
        printf("Memory not allocated!\n");
        return 1;
    }

    // Assign values
    for(int i = 0; i < 5; i++) {
        ptr[i] = i + 1;
    }

    // Print values
    for(int i = 0; i < 5; i++) {
        printf("%d ", ptr[i]);
    }

    free(ptr); // Free memory
    return 0;
}

Common Issues

  • Dangling Pointers: Accessing memory after it is freed.
  • Memory Leaks: Forgetting to free allocated memory.
  • Buffer Overflow: Writing outside allocated memory.
Best Practice: Always check if memory allocation succeeded and free memory when no longer needed.
Conclusion:

Memory management is a crucial concept in C. Proper allocation and deallocation help in building efficient and bug-free programs. Mismanagement can lead to crashes, leaks, and undefined behavior.
