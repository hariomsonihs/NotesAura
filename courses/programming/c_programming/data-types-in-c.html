Data Types in C Programming Language

Introduction to Data Types

Data types in C define the type of data that a variable can hold, including the size, range, and operations that can be performed on it.

  • Definition: A data type specifies the type of value a variable can store, such as integers, floating-point numbers, or characters.
  • Key Points:
    • Data types determine memory allocation and value range.
    • C categorizes data types into basic, derived, enumeration, and void types.
    • Understanding data types is crucial for efficient memory usage and preventing errors.

Basic Data Types

Basic data types are the fundamental building blocks in C, including integers, floating-point numbers, and characters.

  • Definition: Primary data types provided by the C language without any modifiers.
  • Key Points:
    • int: Integer type, typically 4 bytes, range: -2^31 to 2^31-1 (signed).
    • float: Single-precision floating-point, 4 bytes, approximate range: 1.2E-38 to 3.4E+38.
    • double: Double-precision floating-point, 8 bytes, approximate range: 2.3E-308 to 1.7E+308.
    • char: Character type, 1 byte, range: -128 to 127 (signed) or 0 to 255 (unsigned).

Example: Basic Data Types

#include <stdio.h> // Preprocessor directive

int main() {
    int num = 42; // Integer
    float pi = 3.14f; // Float
    double e = 2.71828; // Double
    char letter = 'A'; // Character
    printf("Int: %d, Float: %.2f, Double: %.5f, Char: %c\\n", num, pi, e, letter);
    return 0;
}
  • int num: Stores whole numbers.
  • float pi: Stores single-precision decimals.
  • double e: Stores double-precision decimals.
  • char letter: Stores single characters.

Data Type Modifiers

Modifiers alter the properties of basic data types, such as size, sign, and range.

  • Definition: Keywords like signed, unsigned, short, long that modify basic data types.
  • Key Points:
    • signed: Allows negative values (default for int, char).
    • unsigned: Only non-negative values, doubles positive range.
    • short: Reduces size (e.g., short int: 2 bytes).
    • long: Increases size (e.g., long int: 4 or 8 bytes, long double: 10-16 bytes).

Example: Modified Data Types

#include <stdio.h>

int main() {
    unsigned int uNum = 4294967295; // Max unsigned int
    short int sNum = -32768; // Min short int
    long long int lNum = 9223372036854775807LL; // Max long long
    printf("Unsigned: %u, Short: %hd, Long Long: %lld\\n", uNum, sNum, lNum);
    return 0;
}
  • unsigned int: Positive integers only.
  • short int: Smaller integer range.
  • long long int: Larger integer range.
Note: The exact size of data types can vary by system architecture (e.g., 32-bit vs. 64-bit).

Derived Data Types

Derived data types are built from basic types, including arrays, pointers, structures, and unions.

  • Definition: Complex types derived from primary data types.
  • Key Points:
    • Arrays: Collections of elements of the same type.
    • Pointers: Variables storing memory addresses.
    • Structures: User-defined types grouping different data types.
    • Unions: Like structures but share the same memory location.

Example: Arrays and Pointers

#include <stdio.h>

int main() {
    int arr[3] = {1, 2, 3}; // Array
    int *ptr = &arr[0]; // Pointer to first element
    printf("Array[0]: %d, Via Pointer: %d\\n", arr[0], *ptr);
    return 0;
}
  • int arr[3]: Array of integers.
  • int *ptr: Pointer to integer.

Example: Structures and Unions

#include <stdio.h>

struct Person {
    char name[20];
    int age;
};

union Data {
    int i;
    float f;
};

int main() {
    struct Person p = {"John", 30};
    union Data d;
    d.i = 10;
    printf("Name: %s, Age: %d, Union Int: %d\\n", p.name, p.age, d.i);
    d.f = 3.14f;
    printf("Union Float: %.2f\\n", d.f);
    return 0;
}
  • struct Person: Groups different types.
  • union Data: Shares memory for different types.

Enumeration Data Type

Enumeration (enum) defines a set of named integer constants.

  • Definition: User-defined type consisting of integral constants.
  • Key Points:
    • Improves code readability by using symbolic names for values.
    • By default, starts from 0 and increments by 1.
    • Can assign custom values.

Example: Enumeration

#include <stdio.h>

enum Days {SUN, MON, TUE, WED, THU, FRI, SAT};

int main() {
    enum Days today = WED;
    printf("Today: %d\\n", today); // Outputs 3
    return 0;
}
  • enum Days: Defines constants SUN=0, MON=1, etc.

Void Data Type

The void type represents no value or empty type.

  • Definition: Specifies that no value is available.
  • Key Points:
    • Used in functions that return nothing (e.g., void func()).
    • Void pointers can point to any data type.
    • Cannot declare variables of type void.

Example: Void Type

#include <stdio.h>

void printMessage() {
    printf("Hello, Void!\\n");
}

int main() {
    void *ptr;
    int num = 10;
    ptr = #
    printMessage();
    printf("Value via void pointer: %d\\n", *(int*)ptr);
    return 0;
}
  • void printMessage: Function returns nothing.
  • void *ptr: Generic pointer.

Typedef and Type Qualifiers

Typedef creates aliases for existing types, while qualifiers like const and volatile modify behavior.

  • Definition: Typedef renames data types; qualifiers restrict or specify variable properties.
  • Key Points:
    • typedef: Creates user-defined type names (e.g., typedef int Integer;).
    • const: Makes variables unmodifiable.
    • volatile: Indicates variables can change unexpectedly (e.g., by hardware).
    • restrict: Optimizes pointers by assuming no aliasing (C99).

Example: Typedef and Qualifiers

#include <stdio.h>

typedef unsigned long ulong;

int main() {
    const int constant = 100; // Cannot be changed
    volatile int hardwareVar = 0; // May change externally
    ulong bigNum = 18446744073709551615UL;
    printf("Const: %d, Volatile: %d, Typedef: %lu\\n", constant, hardwareVar, bigNum);
    return 0;
}
  • typedef unsigned long ulong: Alias for unsigned long.
  • const int: Immutable value.
  • volatile int: Prevents optimization assumptions.

Size and Range of Data Types

Use sizeof operator to determine memory size; ranges depend on type and modifiers.

  • Definition: Size is memory bytes; range is min-max values.
  • Key Points:
    • Use <limits.h> and <float.h> for min/max constants.
    • sizeof(type) returns size in bytes.
    • Platform-dependent; use portable types from <stdint.h> (e.g., int32_t).

Example: Size and Range

#include <stdio.h>
#include <limits.h>

int main() {
    printf("Size of int: %zu bytes\\n", sizeof(int));
    printf("Int Min: %d, Max: %d\\n", INT_MIN, INT_MAX);
    printf("Size of double: %zu bytes\\n", sizeof(double));
    return 0;
}
  • sizeof(int): Returns size of int.
  • INT_MIN, INT_MAX: Range constants from limits.h.
