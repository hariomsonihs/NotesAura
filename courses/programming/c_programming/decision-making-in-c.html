Decision Making in C Programming Language

Introduction to Decision Making

Decision making in C allows programs to execute different code blocks based on specific conditions. It is a fundamental concept that enables conditional execution, making programs dynamic and responsive to input or state.

  • Definition: Decision making involves evaluating conditions and executing code blocks based on whether those conditions are true or false.
  • Key Points:
    • Uses conditional statements to control program flow.
    • Relies on relational and logical operators to evaluate conditions.
    • Essential for creating logic-driven applications.

Types of Decision-Making Statements

C provides several constructs for decision making, each suited for different scenarios.

  • Definition: Decision-making statements are constructs that evaluate conditions and control the execution of code blocks.
  • Types:
    • if: Executes a block if a condition is true.
    • if-else: Executes one block if true, another if false.
    • else if: Handles multiple conditions sequentially.
    • switch: Selects a block based on a variable’s value.
    • ?: (Ternary Operator): A concise conditional expression.
Note: Choose the appropriate decision-making construct based on the complexity of conditions and readability requirements.

The if Statement

The if statement evaluates a condition and executes a block of code if the condition is true.

  • Syntax: 
    if (condition) {
    // Code to execute if condition is true
}
  • Key Points:
    • Condition must evaluate to a boolean (0 for false, non-zero for true).
    • Braces are optional for a single statement but recommended for clarity.
    • Can be used with relational/logical operators (e.g., ==, &&).

Example: if Statement

#include <stdio.h>
int main() {
    int num = 10;
    if (num > 0) {
        printf("Number is positive\\n");
    }
    return 0;
}
  • if: Checks if num is greater than 0.
  • Output: "Number is positive" if condition is true.

The if-else Statement

The if-else statement provides an alternative block of code to execute when the condition is false.

  • Syntax: 
    if (condition) {
    // Code if condition is true
} else {
    // Code if condition is false
}
  • Key Points:
    • Only one block (if or else) is executed.
    • Useful for binary decisions (e.g., true/false, yes/no).
    • Improves code readability by handling both outcomes.

Example: if-else Statement

#include <stdio.h>
int main() {
    int num = -5;
    if (num >= 0) {
        printf("Number is non-negative\\n");
    } else {
        printf("Number is negative\\n");
    }
    return 0;
}
  • if-else: Executes else block since num is negative.
  • Output: "Number is negative".

The else if Ladder

The else if construct allows checking multiple conditions sequentially.

  • Syntax: 
    if (condition1) {
    // Code if condition1 is true
} else if (condition2) {
    // Code if condition2 is true
} else {
    // Code if all conditions are false
}
  • Key Points:
    • Conditions are evaluated in order; the first true condition’s block is executed.
    • Optional else block handles cases where no condition is true.
    • Suitable for multiple mutually exclusive conditions.

Example: else if Ladder

#include <stdio.h>
int main() {
    int score = 85;
    if (score >= 90) {
        printf("Grade: A\\n");
    } else if (score >= 80) {
        printf("Grade: B\\n");
    } else if (score >= 70) {
        printf("Grade: C\\n");
    } else {
        printf("Grade: F\\n");
    }
    return 0;
}
  • else if: Checks conditions sequentially.
  • Output: "Grade: B" since score is 85.

The switch Statement

The switch statement selects a code block based on the value of an expression.

  • Syntax: 
    switch (expression) {
    case value1:
        // Code for value1
        break;
    case value2:
        // Code for value2
        break;
    default:
        // Code if no case matches
}
  • Key Points:
    • Expression must evaluate to an integer or character type.
    • break prevents fall-through to subsequent cases.
    • default handles cases where no case matches.
    • More efficient than multiple else if for discrete values.

Example: switch Statement

#include <stdio.h>
int main() {
    char grade = 'B';
    switch (grade) {
        case 'A':
            printf("Excellent\\n");
            break;
        case 'B':
            printf("Good\\n");
            break;
        case 'C':
            printf("Average\\n");
            break;
        default:
            printf("Invalid grade\\n");
    }
    return 0;
}
  • switch: Matches grade to a case.
  • break: Exits switch after executing case.
  • Output: "Good" for grade 'B'.

The Ternary Operator (?:)

The ternary operator is a concise alternative to if-else for simple conditional assignments.

  • Syntax: 
    variable = (condition) ? expression1 : expression2;
  • Key Points:
    • Evaluates condition; assigns expression1 if true, expression2 if false.
    • Best for simple, single-expression conditions.
    • Avoid overuse in complex scenarios for readability.

Example: Ternary Operator

#include <stdio.h>
int main() {
    int a = 10, b = 20;
    int max = (a > b) ? a : b;
    printf("Maximum: %d\\n", max);
    return 0;
}
  • ?: Assigns b to max since a > b is false.
  • Output: "Maximum: 20".

Nested Decision Making

Nested decision-making statements involve placing one decision-making construct inside another.

  • Definition: Nesting occurs when an if, switch, or other construct is used within another decision-making block.
  • Key Points:
    • Allows complex logic by combining multiple conditions.
    • Can reduce readability if overused; keep nesting shallow when possible.
    • Ensure proper indentation to maintain clarity.

Example: Nested if Statements

#include <stdio.h>
int main() {
    int num = 15;
    if (num > 0) {
        printf("Number is positive\\n");
        if (num % 2 == 0) {
            printf("Number is even\\n");
        } else {
            printf("Number is odd\\n");
        }
    }
    return 0;
}
  • Nested if: Checks if num is even/odd within the positive check.
  • Output: "Number is positive" and "Number is odd".

Best Practices for Decision Making

Effective use of decision-making constructs improves code readability and maintainability.

  • Definition: Best practices guide the use of decision-making statements to ensure clarity and efficiency.
  • Key Points:
    • Use clear and meaningful conditions to improve readability.
    • Prefer switch over multiple else if for discrete values.
    • Avoid deep nesting; consider breaking complex logic into functions.
    • Always include break in switch cases unless fall-through is intentional.
    • Use the ternary operator for simple assignments, not complex logic.
