Identifiers in C

What are Identifiers?

In C, identifiers are the names used to identify variables, functions, arrays, structures, or any other user-defined elements. They act as symbolic names that represent memory locations or code elements.

  • Identifiers make programs readable and meaningful.
  • Examples: age, sum, calculateArea, student_name.

Rules for Naming Identifiers

  • Must begin with a letter or underscore (_).
  • Subsequent characters may include letters, digits (0–9), or underscores.
  • No special symbols allowed (e.g., @, $, %).
  • Cannot be a C keyword (e.g., int, while).
  • Case-sensitive (value ≠ Value).
  • No length limit in C standard, but compilers may restrict (often 31 characters).

Valid and Invalid Identifiers

int age;          // valid
float totalMarks;  // valid
char _grade;       // valid

int 123value;     // invalid (starts with digit)
float total%marks; // invalid (special symbol % not allowed)
char while;        // invalid (keyword)

Identifiers vs Keywords

Keywords are reserved words in C (like int, if, return) that have predefined meaning. Identifiers are user-defined names that programmers create.

  • Keyword: int (defines an integer type).
  • Identifier: marks (user-defined variable name).
Tip: Never use keywords as identifiers. It causes compilation errors.

Code Example

#include <stdio.h>

int main() {
    int age = 20;
    float marks = 89.5;
    char grade = 'A';

    printf("Age: %d\n", age);
    printf("Marks: %.2f\n", marks);
    printf("Grade: %c\n", grade);

    return 0;
}

Common Mistakes

  • Using keywords as identifiers (e.g., int float;).
  • Starting identifiers with digits (2value).
  • Including spaces (student name).
  • Mixing case unintentionally (total vs Total).
Conclusion:

Identifiers are fundamental to C programming, providing readable names for data and functions. Following proper naming rules ensures clarity, avoids conflicts with keywords, and makes code easier to maintain. Always use meaningful identifiers like studentAge or calculateSum for better readability.
