Identifiers in C

In C programming, identifiers are the names given to variables, functions, arrays, structures, and other user-defined elements. They allow programmers to reference and manipulate data stored in memory in a human-readable way.

Rules for Naming Identifiers

  • Identifiers can contain letters (A–Z, a–z), digits (0–9), and underscores (_).
  • They must begin with a letter or an underscore, not a digit.
  • C is case-sensitive, so Var and var are different identifiers.
  • Keywords (like int, for, return) cannot be used as identifiers.
  • No special symbols (like @, $, -) are allowed.
  • Although no strict limit, many compilers consider only the first 31 characters significant.

Examples of Identifiers

/* Valid identifiers */
int age;
float total_marks;
char studentName[30];

/* Invalid identifiers */
int 2ndNumber;   // ❌ starts with digit
float total$;    // ❌ contains invalid symbol
char break;      // ❌ keyword used as identifier

Identifiers vs Keywords

Identifiers: User-defined names for variables, functions, etc. Example: age, totalMarks, sum1

Keywords: Predefined reserved words in C with specific meanings. Example: int, while, return

Example Program

#include <stdio.h>
int main() {
    int number = 10;        // 'number' is an identifier
    float average = 89.5;   // 'average' is an identifier
    printf("Number: %d, Average: %.2f", number, average);
    return 0;
}

Common Mistakes

  • Using keywords as identifiers (e.g., int int; ❌).
  • Starting identifiers with digits (e.g., 1value ❌).
  • Using invalid characters like @, $, - ❌.
  • Mixing inconsistent styles (camelCase in one place, snake_case in another).

Best Practices

  • Use meaningful names (e.g., totalMarks instead of tm).
  • Follow consistent naming conventions (camelCase or snake_case).
  • Avoid overly long identifiers, but make them descriptive.

Conclusion

Identifiers in C are fundamental for writing clear, understandable, and error-free programs. By following the rules and best practices, you can ensure your code is both correct and readable.
