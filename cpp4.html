Object-Oriented Programming in C++

Explore the power of object-oriented programming (OOP) in C++, a paradigm that enhances modularity, reusability, and scalability through classes, objects, and more.

Classes and Objects

Classes are blueprints for objects, defining properties and behaviors. Objects are instances of classes, bringing the blueprint to life. 


#include 
using namespace std;

class Car {
public:
    string brand;
    int speed;
    void display() {
        cout << brand << " is moving at " << speed << " km/h" << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota";
    car1.speed = 120;
    car1.display();
    return 0;
}

Explanation

  • class Car: Defines a class with properties (brand, speed) and a method (display).
  • Car car1: Creates an object of the Car class.
  • car1.brand: Sets object properties.
  • car1.display(): Calls the objectâ€™s method.

Inheritance

Inheritance allows a class to inherit properties and methods from another class, promoting code reuse and hierarchy. 


#include 
using namespace std;

class Vehicle {
public:
    string type = "Vehicle";
    void move() {
        cout << type << " is moving" << endl;
    }
};

class Bike : public Vehicle {
public:
    Bike() {
        type = "Bike";
    }
};

int main() {
    Bike bike;
    bike.move();
    return 0;
}

Explanation

  • class Vehicle: Base class with a property and method.
  • class Bike : public Vehicle: Derived class inherits from Vehicle.
  • bike.move(): Calls inherited method with modified type.

Polymorphism

Polymorphism allows methods to behave differently based on the object calling them, achieved through virtual functions and overriding. 


#include 
using namespace std;

class Animal {
public:
    virtual void sound() {
        cout << "Some sound" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() override {
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* animal = new Cat();
    animal->sound();
    delete animal;
    return 0;
}

Explanation

  • virtual void sound(): Declares a virtual function in the base class.
  • void sound() override: Overrides the method in the derived class.
  • Animal* animal = new Cat(): Uses a base class pointer to a derived class object.
  • animal->sound(): Calls the overridden method (runtime polymorphism).

Encapsulation

Encapsulation hides data and exposes only necessary parts through public methods, ensuring data integrity and security. 


#include 
using namespace std;

class BankAccount {
private:
    double balance;
public:
    BankAccount() : balance(0.0) {}
    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }
    double getBalance() {
        return balance;
    }
};

int main() {
    BankAccount account;
    account.deposit(1000.50);
    cout << "Balance: $" << account.getBalance() << endl;
    return 0;
}

Explanation

  • private: double balance: Hides data from external access.
  • public: void deposit: Provides controlled access to modify data.
  • getBalance(): Allows safe retrieval of data.

Abstraction

Abstraction simplifies complex systems by exposing only essential features, often implemented using abstract classes or interfaces. 


#include 
using namespace std;

class Shape {
public:
    virtual double area() = 0; // Pure virtual function
    virtual ~Shape() {}
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() override {
        return 3.14159 * radius * radius;
    }
};

int main() {
    Circle circle(5.0);
    cout << "Circle area: " << circle.area() << endl;
    return 0;
}

Explanation

  • virtual double area() = 0: Defines a pure virtual function, making Shape abstract.
  • class Circle: Implements the abstract method.
  • circle.area(): Calls the implemented method.

Conclusion

Object-oriented programming in C++ transforms the way you design software, offering modularity, reusability, and maintainability through classes, inheritance, polymorphism, encapsulation, and abstraction. By mastering these OOP principles, you can build robust, scalable applications, from simple utilities to complex systems.
