Basic HTML Structure

The foundational structure of an HTML document.

  • <!DOCTYPE html>: Declares the document as HTML5.
  • <html>: Root element of the HTML page.
  • <head>: Contains metadata, links to CSS, and scripts.
  • <body>: Contains the visible content of the page.

Example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page</title>
</head>
<body>
    <h1>Welcome</h1>
</body>
</html>

Tip: Always include lang="en" in the <html> tag for accessibility.

HTML Comments

Comments are used to add notes in the code that are not displayed in the browser.

  • <!-- Comment -->: Syntax for adding comments.
  • Comments can span multiple lines.
  • Useful for debugging or explaining code sections.

Example:

<!-- This is a comment -->
<p>This is visible text.</p>
<!-- <p>This is hidden text.</p> -->

Trick: Use comments to temporarily disable code during testing.

Inline and Internal Styles

Apply CSS styles directly in HTML for quick styling.

  • style attribute: Inline CSS applied directly to an element.
  • <style>: Internal CSS defined in the <head>.
  • Inline styles override internal/external styles.

Example:

<!-- Inline CSS -->
<p style="color: blue; font-size: 16px;">Blue text</p>

<!-- Internal CSS -->
<head>
    <style>
        p { color: red; }
    </style>
</head>

Tip: Prefer external CSS for maintainability, use inline for quick tests.

Quotations

Tags for quoting text or citing sources.

  • <blockquote>: Long quotations, often indented.
  • <q>: Short inline quotations.
  • <cite>: Reference to a source or work.
  • <abbr>: Abbreviations with a title attribute.

Example:

<blockquote cite="https://example.com">
    <p>A long quote here.</p>
</blockquote>
<p>He said, <q>Hello!</q></p>
<cite>Book Title</cite>
<abbr title="World Health Organization">WHO</abbr>

Tip: Use cite attribute in <blockquote> to link to the source.

Class and ID Attributes

Attributes to identify and style elements.

  • class: Assigns multiple elements to a group for styling or scripting.
  • id: Unique identifier for a single element.

Example:

<div class="container">Content</div>
<div id="header">Header</div>

Fact: id must be unique per page, while class can be reused.

Marquee

Creates scrolling text (deprecated, use CSS animations instead).

  • <marquee>: Scrolls text horizontally or vertically.
  • direction: Sets scroll direction (left, right, up, down).
  • behavior: Sets scroll type (scroll, slide, alternate).

Example:

<marquee direction="left" behavior="scroll">Scrolling text</marquee>

Tip: Avoid <marquee> in modern web design; use CSS @keyframes instead.

Meta Tags

Metadata for defining document properties and behavior.

Tag Description Example
<meta charset>Sets character encoding<meta charset="UTF-8">
<meta name="viewport">Controls viewport for responsive design<meta name="viewport" content="width=device-width">
<meta name="description">Page description for SEO<meta name="description" content="Page info">
<meta name="keywords">Keywords for search engines<meta name="keywords" content="html, css">
<meta name="author">Author of the page<meta name="author" content="Name">

Example:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Learn HTML">
</head>

Tip: Always include charset and viewport meta tags for compatibility.

Lists

Organize content with ordered and unordered lists.

  • <ul>: Unordered list (bulleted).
  • <ol>: Ordered list (numbered).
  • <li>: List item within <ul> or <ol>.
  • <dl>, <dt>, <dd>: Description list for terms and definitions.

Example:

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>
<ol>
    <li>Step 1</li>
    <li>Step 2</li>
</ol>
<dl>
    <dt>Term</dt>
    <dd>Definition</dd>
</dl>

Fact: Nested lists can create multi-level menus.

Links and Navigation

Create hyperlinks for navigation.

  • <a href="URL">: Anchor tag for links.
  • target="_blank": Opens link in a new tab.
  • rel="nofollow": Prevents search engine crawling.

Example:

<a href="https://example.com" target="_blank">Visit Example</a>

Tip: Use relative URLs for internal links to improve portability.

Images and Media

Embed images, videos, and audio.

  • <img src="path" alt="description">: Embeds an image.
  • <video>: Embeds video content.
  • <audio>: Embeds audio content.
  • <figure> & <figcaption>: Groups media with captions.

Example:

<img src="image.jpg" alt="Description" width="300">
<video controls>
    <source src="video.mp4" type="video/mp4">
</video>
<figure>
    <img src="image.jpg" alt="Description">
    <figcaption>Image caption</figcaption>
</figure>

Trick: Always include alt attributes for accessibility and SEO.

Forms

Create interactive forms for user input.

  • <form>: Container for form elements.
  • <input>: Input fields (text, email, password, etc.).
  • <label>: Labels for input fields.
  • <select> & <option>: Dropdown menus.
  • <textarea>: Multi-line text input.
  • <button>: Submit or reset buttons.

Input Types:

Type Description Example
textSingle-line text input<input type="text">
passwordHidden text input<input type="password">
emailEmail input with validation<input type="email">
numberNumeric input<input type="number">
checkboxMultiple selection<input type="checkbox">
radioSingle selection<input type="radio">
fileFile upload<input type="file">
dateDate picker<input type="date">

Example:

<form action="/submit">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    <select name="choice">
        <option value="1">Option 1</option>
    </select>
    <textarea name="message"></textarea>
    <button type="submit">Submit</button>
</form>

Tip: Use required attribute to enforce input validation.

Tables

Display data in a tabular format.

  • <table>: Table container.
  • <tr>: Table row.
  • <th>: Table header cell.
  • <td>: Table data cell.
  • <caption>: Table caption.

Example:

<table>
    <caption>Data Table</caption>
    <tr>
        <th>Header</th>
        <th>Header</th>
    </tr>
    <tr>
        <td>Data</td>
        <td>Data</td>
    </tr>
</table>

Fact: Use scope="col" or scope="row" in <th> for accessibility.

Semantic HTML

Use tags that convey meaning and structure.

  • <header>: Page or section header.
  • <nav>: Navigation links.
  • <main>: Main content area.
  • <footer>: Page or section footer.
  • <article>: Independent content block.
  • <section>: Thematic content grouping.
  • <aside>: Sidebar or supplementary content.

Example:

<header>
    <nav>...</nav>
</header>
<main>
    <article>...</article>
    <aside>Sidebar</aside>
</main>
<footer>...</footer>

Trick: Semantic tags improve SEO and accessibility.

Advanced HTML5 Elements

HTML5-specific elements for modern web development.

  • <canvas>: For drawing graphics via JavaScript.
  • <svg>: Scalable vector graphics.
  • <details> & <summary>: Collapsible content.
  • <picture>: Responsive image sources.
  • <datalist>: Autocomplete options for inputs.

Example:

<details>
    <summary>More Info</summary>
    <p>Hidden content here.</p>
</details>
<input list="options">
<datalist id="options">
    <option value="Option 1">
    <option value="Option 2">
</datalist>

Tip: Use <picture> for responsive images with <source> tags.

Accessibility (ARIA)

Enhance accessibility with ARIA attributes.

  • role: Defines element role (e.g., role="button").
  • aria-label: Provides accessible name for elements.
  • aria-hidden: Hides elements from screen readers.
  • aria-live: Indicates live content updates.

Example:

<button aria-label="Close">X</button>
<div aria-live="polite">Updated content</div>

Fact: ARIA attributes improve usability for screen reader users.
